Benchmark surface

Semantic application form

This page is intentionally structured like a realistic recruiting workflow: required contact fields, repeated Yes / No questions, a location chooser, an opt-in toggle, and longer essay responses. It is designed for comparing Geometra MCP semantic tools against a Playwright snapshot plus script flow.

Agents should be able to infer the form shape quickly, fill it reliably by label, and confirm that no required fields remain invalid without resorting to large full-page dumps.

Written responses

These long-form prompts are here to make the benchmark closer to a real application flow where the agent has to move beyond tiny text fields and keep payloads under control.

What this benchmark should highlight

  • Whether a summary-first page model is smaller than a full accessibility snapshot.
  • Whether labeled field filling reduces the need to emit a large custom browser script.
  • Whether repeated Yes / No groups and required-field validation can be handled generically.
